Testing pomodoro timer API

Users

  • Note*: public profile = no credentials data
  • ../api/users : get all users public profiles (for leaderboard)
  • ../api/users/{id} : get user with that id (public profile)
  • ../api/users/login : pass in a user JSON through POST method (username and password is a must) to login and return the user credentials data and task list

    Example: {"id":"SE160947","name":"testing@test.com","password":"123456","timeFocusToday":13575,"timeFocusThisWeek":794850}

    Note* : return status : 601 = ERROR ; 201 = SUCCESS ; 602 = INVALID_CREDENTIALS
  • ../api/users/create : pass in new user JSON through POST method to create new user (there's a dublicate check and userid length must >= 8)

    Example: {"id":"SE160947","name":"testing@test.com","password":"123456","timeFocusToday":13575,"timeFocusThisWeek":794850}

    Note* : return status : 601 = ERROR ; 201 = SUCCESS ; 602 = user_errors(DUPLICATE / INVALID_USER_ID_LENGTH)
  • ../api/users/update : pass in user JSON through POST method to update that userId

    Example: {"id":"PTU00007","name":"testingUpdate@test.com","password":"123456","timeFocusToday":0,"timeFocusThisWeek":0}

    Note* : return status : 601 = ERROR ; 200 = OK ; 602 = USER_NOT_FOUND
  • ../api/users/{userId}/updateTask : pass in task JSON through POST method to update task for that userId

    Example: {"id":17,"userId":"SE160947","name":"OJT SignUp2","duration":1209600,"startTime":1660755600000,"completeTime":1661965200000,"status":1}

    Note* : return status : 601 = ERROR ; 200 = OK ; 602 = user_errors(USER_NOT_FOUND / TASK_NOT_FOUND)
  • ../api/users/{userId}/addTask : delete that user (DELETE method)
    Note* : return status : 601 = ERROR ; 200 = OK ; 602 = USER_NOT_FOUND
  • ../api/users/{userId}/addTask : delete that user's task (DELETE method)
    Note* : return status : 601 = ERROR ; 200 = OK ; 602 = user_errors(USER_NOT_FOUND / TASK_NOT_FOUND)

Custom return error codes

  • 601 : server query error
  • 602 : User/FrontEnd input error
